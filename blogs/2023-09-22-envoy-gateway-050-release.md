<h4>Guest Author</h4><h4>All you need to know</h4><p>This is the third article in our<a href="https://blog.getambassador.io/an-overview-of-the-kubernetes-gateway-api-and-envoy-gateway-8162bfb7fb9e"> ongoing series</a> on<a href="https://gateway.envoyproxy.io/"> Envoy Gateway</a> releases, covering the July 2023 release of<a href="https://github.com/envoyproxy/gateway/releases/tag/v0.5.0"> v0.5.0</a>. Our goal with this series is not just to rehash the <a href="https://gateway.envoyproxy.io/v0.5.0/releases/v0.5.html">release notes</a> or cover them in full but to highlight major and minor changes we think are important and provide context. This release includes:</p><ul><li>Support for data plane proxy telemetry</li><li>Support for the Envoy Patch Policy API</li><li>Admin server for Envoy Gateway</li><li>Added best practices default edge settings to xDS resources</li></ul><p>If you need a refresher on what Envoy Gateway is and the project goals, check out our previous post on the <a href="https://blog.getambassador.io/envoy-gateway-0-4-0-release-8556b3291489">v0.4.0 release</a>.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*oXAfDxc83AJ-MOPhGJtbBg.png" /></figure><h4><strong>Improvements to the documentation and installation process</strong></h4><p>With the release of v0.5.0, the documentation has been updated to include necessary instructions for the following topics:</p><ul><li>Installation using the Helm package manager</li><li>Customization of the Helm charts</li><li>Usage of cert-manager for TLS termination</li><li>Documentation on the presentation links</li><li>Documentation on the configuration of multiple TLS certificates per listener</li></ul><p>These additions to documentation make getting started and working with Envoy Gateway easier. Apart from the documentation, there have also been improvements to the installation process. Some of them are:</p><ul><li>Support for configuring Envoy Gateway label and annotations using Helm</li><li>Increase in default resource defaults for Envoy Gateway to 100m CPU and 256Mi memory</li><li>Possibility for users to bring their own TLS certificates by adding an opt-in field to skip certificate creation for the control plane</li></ul><h4><strong>New and exciting API features</strong></h4><p>The API has probably received the biggest number of new features and improvements in this release, which include:</p><ul><li><strong>Support for Envoy Patch Policy: </strong>This release has finally brought the support for Envoy Patch Policy to the Envoy Gateway, allowing advanced users to be able to leverage Envoy Proxy functionality not exposed by Envoy Gateway APIs. The users can now modify the output xDS configuration generated by Envoy Gateway using JSON patch semantics.</li><li><strong>Support for Envoy Proxy Telemetry: </strong>Users can now keep an eye on the Control Plane as well as the underlying Envoy Proxy instances. For collecting logs from the Envoy Proxy instance, FluentBit is used, and the collected logs are then forwarded to Loki for storage whileTraces are stored using Tempo. Finally, OpenTelemetry is used as a means to receive, process, and export the telemetry data, allowing the users to have observability over logging, traces, and metrics.</li><li><strong>Addition of admin server for Envoy Gateway:</strong> With the improved installation process, the admin server is now available at port 19000 of the local host by default, allowing easy management of the different aspects of the server.</li><li><strong>Support for Envoy Proxy and rate limiting specific configuration options:</strong> This new release allows users to configure Envoy Proxy Pod labels, deployment strategy settings, volumes, and volume mounts. This release also allows users to configure envoy proxy as a NodePort type server. Distinct <a href="https://www.getambassador.io/blog/rate-limiting-apis-scale-patterns-strategies">rate limiting</a> for different IP addresses is now possible, as well as the conversion of JWT claims to headers to be used for Rate Limiting.</li></ul><h4><strong>The next Envoy Gateway release</strong></h4><p>The next release of Kubernetes Gateway API will be <a href="https://gateway.envoyproxy.io/v0.5.0/design/roadmap.html">v0.6.0</a> with a theme of “Preparation for GA.” The majority of work being done is visible in the <a href="https://github.com/envoyproxy/gateway/milestone/15">0.6.0-rc.1 milestone</a> on GitHub. We’ll provide an update when that release is available!</p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=ccb4f1ca33e7" width="1" /><hr /><p><a href="https://blog.getambassador.io/envoy-gateway-0-5-0-release-ccb4f1ca33e7">Envoy Gateway 0.5.0 Release</a> was originally published in <a href="https://blog.getambassador.io">Ambassador Labs</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>

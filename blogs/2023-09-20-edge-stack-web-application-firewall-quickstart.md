<p>One-Stop API Gateway solution with built-in PCI 6.6 compliant WAF.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/960/1*N89VVZmlL6yOdthiY7G6FA.png" /><figcaption>Edge Stack WAF — One-stop built-in security</figcaption></figure><p>In today’s landscape of growing security and compliance concerns, finding the right solution to open up your application to the world can be time consuming and involve sourcing bits of the solution from multiple vendors. Ambassador Labs is here to help with our Ambassador Edge Stack API Gateway now with its own WAF functionality built in. No longer do you need to pair a potentially expensive third party WAF offering with your API gateway to meet your company’s compliance needs.</p><p>We make getting started opening your application up for the world while keeping a strong security posture simple. By applying two convenient CRDs to your running AES installation, you can quickly enable the Ambassador Labs WAF. We even provide you with a packaged set of rules that offer OWASP Top 10 protection and PCI 6.6 compliance to help you get started. These rules can be further customized to meet your needs or you can bring in your own rule set built with <a href="https://coraza.io/docs/seclang/directives">Coraza’s SecLang</a>.</p><p>Ambassador Edge Stack provides a one-stop solution to open access to your application with strong auth controls and integrations, <a href="https://www.getambassador.io/blog/rate-limiting-apis-scale-patterns-strategies">rate limiting</a>, and now WAF protection. To see just how easy it is to get started, let’s go through setting up a new cluster and protecting it with the WAF.</p><h3>Installing Ambassador Edge Stack into your cluster</h3><p>The best way to quickly get started with installing AES is to follow our <a href="https://www.getambassador.io/docs/edge-stack/latest/tutorials/getting-started">QuickStart</a> guide. This will get you up and running with a basic installation of Edge Stack complete with a service that you can use to test the cluster.</p><h3>Assign a hostname to your listener</h3><p>Use whatever method is used for your provider of choice for adding a DNS entry for your listener. For a simple test, you can add the IP address held in $LB_ENDPOINT from Step 1 to your Hosts file (/etc/hosts) if on a *nix system.</p><pre>.<br />.<br />.<br />&lt;IP ADDRESS&gt;  waf-blog.localdomain</pre><h3>Enabling the WAF</h3><p>Now that we have AES up and running, let’s get to the good part: enabling the WAF to process your incoming traffic.</p><h4>Creating a new WebApplicationFirewall Resource</h4><p>The first step is to create a WebApplicationFirewall resource. This resource configures a firewall instance with a set of rules and runtime options, like if request denials should be logged or not. Rules can be stored at HTTP endpoints or in ConfigMaps. Our starting resource will use a ruleset created by Ambassador Labs and retrieved from HTTP endpoints. This starter rule set configures the WAF to use all of OWASP’s Core rule set, which protects your system from the OWASP Top 10 vulnerabilities, and has been enhanced by us to also provide PCI 6.6 compliance.</p><pre>kubectl apply -f -&lt;&lt;EOF<br />---<br />apiVersion: gateway.getambassador.io/v1alpha1<br />kind: WebApplicationFirewall<br />metadata:<br />  name: &quot;example-waf&quot;<br />  namespace: &quot;default&quot;<br />spec:<br />  firewallRules:<br />    - sourceType: &quot;http&quot;<br />      http:<br />        url: &quot;https://app.getambassador.io/download/waf/v1-20230825/aes-waf.conf&quot;<br />    - sourceType: &quot;http&quot;<br />      http:<br />        url: &quot;https://app.getambassador.io/download/waf/v1-20230825/crs-setup.conf&quot;<br />    - sourceType: &quot;http&quot;<br />      http:<br />        url: &quot;https://app.getambassador.io/download/waf/v1-20230825/waf-rules.conf&quot;<br />EOF</pre><h4>Creating a new WebApplicationFirewallPolicy Resource</h4><p>Next, the WebApplicationFirewallPolicy resource instructs Edge Stack on which requests should be protected by which WebApplicationFirewall resource. Your Policy can reference multiple Firewalls, so you can consolidate all your matching criteria and WAFs into a single Policy. For this example, however, we are creating a Policy that references our single example Firewall, and applies it to all incoming requests (the default if no more specific rules are provided).</p><pre>kubectl apply -f -&lt;&lt;EOF<br />---<br />apiVersion: gateway.getambassador.io/v1alpha1<br />kind: WebApplicationFirewallPolicy<br />metadata:<br />  name: &quot;example-waf-policy&quot;<br />  namespace: &quot;default&quot;<br />spec:<br />  rules:<br />  - wafRef:<br />      name: &quot;example-waf&quot;<br />      namespace: &quot;default&quot;<br />EOF</pre><h4>Making sure everything is ready</h4><pre>kubectl get waf &amp;&amp; kubectl get wafp</pre><h3>Sending some requests</h3><h4>Send a good request</h4><pre>curl -Lki https://&lt;HOSTNAME&gt;/backend/</pre><p>Here, we should see a valid response:</p><pre>HTTP/1.1 200 OK<br />content-type: application/json<br />date: Fri, 08 Sep 2023 17:22:18 GMT<br />content-length: 223<br />x-envoy-upstream-service-time: 0<br />server: envoy<br />{<br />    &quot;server&quot;: &quot;thunderous-banana-mgune5tc&quot;,<br />    &quot;quote&quot;: &quot;The light at the end of the tunnel is interdependent on the relatedness of motivation, subcultures, and management.&quot;,<br />    &quot;time&quot;: &quot;2023-09-08T17:22:18.22809918Z&quot;<br />}</pre><h4>Send a bad request</h4><p>On this request we set the User-Agent to <a href="https://github.com/Arachni/arachni">Arachni</a>, a penetration testing tool. If someone is using Arachni on your system, they are looking for vulnerabilities and you want to fend them off!</p><pre>curl -Lki -H 'User-Agent: Arachni/0.2.1'  https://&lt;HOSTNAME&gt;/backend/</pre><p>Here we should see the request being blocked by the WAF:</p><pre>HTTP/1.1 403 Forbidden<br />date: Fri, 08 Sep 2023 17:22:56 GMT<br />server: envoy<br />content-length: 0</pre><p>Congratulations, you now have a cluster up and running that is protected by a WAF, all handled by Ambassador Edge Stack!</p><h3>Next Steps</h3><p>For a deeper dive into the mechanics of the WAF, and best practices that we followed at Ambassador Labs on our way to SOC2 certification using our own WAF, you can view our recent <a href="https://www.getambassador.io/webinar/waf">webinar</a> on-demand.</p><p>For more information on the options for the Ambassador Edge Stack WAF, please checkout our <a href="https://www.getambassador.io/docs/edge-stack/latest/howtos/web-application-firewalls">WAF user guide</a>.</p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=33c19f3b9489" width="1" /><hr /><p><a href="https://blog.getambassador.io/edge-stack-web-application-firewall-quickstart-33c19f3b9489">Edge Stack Web Application Firewall Quickstart</a> was originally published in <a href="https://blog.getambassador.io">Ambassador Labs</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>
